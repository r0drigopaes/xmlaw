Princípios XMLaw:
(1) Exitem leis que governam a interação entre os agentes de software.
(2) Leis precisam ser aplicadas via software.
(3) As leis precisam ser especificadas explicitamente ao invés de estarem implícitas no código de cada um dos agentes.
(4) Deve ser possível especificar leis de maneira modular, de forma a permitir a sua reutilização.

- DEFINIÇÃO / COMPOSIÇÃO

- Protocolos podem ser definidos externamente em seus próprios arquivos. Assim, podemos utilizar um mecanismo similar ao "import" do java para que possamos reutilizar a definicao do protocolo.

- Os elementos que definem escopo são: Organization e Scene.

- 'Organization' pode conter 'scene' como sub-escopo.

- Dentro de um escopo é possível definir os seguintes elementos: Protocol, Clock, Norm, Action e Constraint.

- Os elementos definidos dentro de um escopo podem ser utilizados no mesmo ou em um de seus sub-escopos.
